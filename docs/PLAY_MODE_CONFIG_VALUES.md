# 대국모드 설정값 일람

수정 반영 후, **높은 승률 수만이 아니라 다양한 수를 탐색**하도록 레벨별로 아래 값이 play.cfg → MCTS에 적용됩니다.

---

## 1. 동작 요약

- **프로(레벨 0)**: 탐색은 좁게(cpuct·노이즈 낮음) → 최선에 가깝게만 둠.
- **레벨 1~30**: 레벨 숫자가 **클수록(약할수록)** cpuct·wideRootNoise가 **커져서** 탐색이 넓어지고, 다양한 수에 방문이 퍼짐 → 후보 수가 많아질 여지가 있음.

---

## 2. 프로(레벨 0) 고정값

| 파라미터 | 값 |
|----------|-----|
| chosenMoveTemperature | 0.0 |
| avoidRepeatedPatternUtility | 0.0 |
| staticScoreUtilityFactor | 0.0 |
| wideRootNoise | 0.0 |
| cpuctExploration | 1.0 |
| cpuctExplorationLog | 0.45 |
| cpuctExplorationBase | 500.0 |
| avoidRepeatedPattern | true (고정) |

→ **다양한 수 탐색 최소**, 항상 최고 승률에 가깝게 선택.

---

## 3. 레벨 1~30 (앵커 보간)

레벨 인덱스 `L`(1~30)에 대해, 앵커 구간별 **선형 보간** 후 표시한 클램프 범위 안으로 적용됩니다.

### 3.1 탐색 폭·다양성 관련 (MCTS에 직접 반영)

| 파라미터 | 앵커 (레벨 → 값) | 적용 범위 | 의미 |
|----------|-------------------|-----------|------|
| **wideRootNoise** | 1→0.03, 10→0.12, 30→0.20 | 0~0.2 | 루트 노이즈. **클수록** 다양한 수 탐색. |
| **cpuctExploration** | 1→0.80, 10→4.50, 30→10.00 | 0~10 | PUCT 탐색. **클수록** 덜 탐색된 수도 더 골라서 방문 분산. |
| **cpuctExplorationLog** | 1→0.80, 10→4.50, 30→10.00 | 0~10 | PUCT 로그 항. **클수록** 탐색 폭 증가. |
| **cpuctExplorationBase** | 1→200, 10→1200, 30→4000 | 10~100000 | PUCT 베이스. |

→ 레벨 **30(최저기력)** 에서 위 값들이 가장 크게 들어가서 **가장 다양한 수를 탐색**합니다.

### 3.2 착수 선택·스타일 관련 (정책/유틸리티)

| 파라미터 | 앵커 (레벨 → 값) | 적용 범위 |
|----------|-------------------|-----------|
| chosenMoveTemperature | 1→0.20, 10→1.10, 30→2.20 | 0.05~2.2 |
| avoidRepeatedPatternUtility | 1→0.20, 10→1.20, 30→1.96 | 0~2.0 |
| staticScoreUtilityFactor | 1→0.20, 10→0.60, 30→0.85 | 0~1.0 |

- avoidRepeatedPattern = **true** 고정.

### 3.3 공통(모든 레벨)

| 파라미터 | 값 |
|----------|-----|
| maxTime (play.cfg) | 30.0 |
| maxVisits | 100000 |
| winLossUtilityFactor | 1.0 |
| dynamicScoreUtilityFactor | 0.3 |

### 3.4 실제 착수당 분석 시간(앱이 요청하는 값)

| 레벨 | 시간 범위 (초) |
|------|----------------|
| 0 (프로) | 3~6 |
| 1~6 (초고수) | 3→2, 6→3 (보간) |
| 7 (고수1) | 1.6~2 |
| 8~30 | 1.6→1, 2→1 (보간) |
| 30 (최저기력) | 1~1 |

## 5. 요약 한 줄

- **다양한 수를 탐색하게 하는 설정**: **wideRootNoise**, **cpuctExploration**, **cpuctExplorationLog**, **cpuctExplorationBase**  
  → 레벨이 낮을수록(숫자가 클수록) 위 값들이 커져서, **높은 승률만이 아니라 다양한 수**를 탐색합니다.
- 위 값들은 `startGame()`에서 레벨별로 계산되어 SettingsRepository → play.cfg에 기록되고,  
  수정된 `analyzeTop4()`에서는 **덮어쓰지 않고** config 값을 그대로 씁니다.
