# "Assets in 'katago/': ..." 로그 설명

## 엔진은 2개뿐입니다

앱이 **실제로 쓰는 모델(엔진)** 은 다음 **2개**만 있습니다.

| 파일 | 용도 |
|------|------|
| **b10.bin.gz** → 압축 해제 후 **b10.bin** | 분석 전용, 대국 레벨 0~6 (프로·초고수) |
| **b6.txt.gz** → 압축 해제 후 **b6.txt** | 대국 레벨 7~30 |

g170은 **사용하지 않습니다.**

---

## 로그에 나오는 목록이 의미하는 것

`Assets in 'katago/': README.txt, analysis.cfg, b10.bin, b10.bin.gz, b6.txt, b6.txt.gz, g170.txt.gz`

이건 **"엔진이 3개다"**가 아니라, **에셋 폴더( katago/ ) 안에 들어 있는 파일 이름을 전부 나열한 것**입니다.

- **README.txt, analysis.cfg**  
  설정/설명용. 엔진 아님.
- **b10.bin.gz, b6.txt.gz**  
  우리가 쓰는 **2개 모델**의 압축 파일. 설치 시 압축 풀어서 b10.bin, b6.txt 로 복사함.
- **b10.bin, b6.txt**  
  빌드/배포 과정에서 압축 해제본이 에셋에 같이 들어가 있으면 목록에 뜰 수 있음. 앱은 **.gz만 골라서** 쓰도록 되어 있음.
- **g170.txt.gz**  
  예전에 넣어 둔 파일이라 **에셋 폴더에만 있고**, 앱 코드는 **전혀 사용하지 않음**. 목록에만 보이고 엔진 개수와 무관함.

정리하면, **엔진은 b10·b6 두 개뿐**이고, 나머지는 설정/설명 파일이거나 사용하지 않는 파일입니다.
