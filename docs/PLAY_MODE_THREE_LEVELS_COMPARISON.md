## 3. 어떤 점이 다른지 (요약)

### 3.1 착수 로직·낮은 승률

- **레벨 30만**  
  - **최저기력** 전용: 후보 중 **최하승률 5%**만 풀에 넣고 그 안에서 무작위.  
  - 로그상 **pool=1** → 국면마다 “최하 5%”에 해당하는 수가 1개뿐.  
  - **낮은 승률 선택 적용** = 의도적으로 가장 나쁜 수에 가깝게 둠.
- **레벨 8·14·28**  
  - **고수1~** 구간: **pickGradualStrength** — 하위 비율 풀에서 승률·방문수 가중치로 샘플링.  
  - **낮은 승률 선택 미적용** (최저 승률만 골라 두는 로직 아님).  
  - **pool**이 5~9로 다양 → 여러 후보 중에서 강도에 맞게 선택.

### 3.2 파라미터 경향

- **레벨이 높을수록(숫자 클수록, 약할수록)**  
  - chosenMoveTemp·wideRootNoise·cpuct·avoidRepeatUtility·staticScoreUtility **증가** → 탐색·다양성·실수 허용 증가.
- **레벨이 낮을수록(숫자 작을수록, 강할수록)**  
  - 위 파라미터 **감소** → best에 가깝게 두는 비율 증가.

### 3.3 방문수·시간

- **제한 시간**  
  - 레벨 30: **1.0s** 고정.  
  - 레벨 14: 1.42~1.70s, 레벨 8: 1.57~1.96s → 레벨이 올라갈수록(강해질수록) 착수당 시간이 조금씩 늘어남.
- **국면당 총 방문수**  
  - 레벨 30: 68~172 (짧은 시간이라 낮은 구간 많음).  
  - 레벨 14: 35~246 (일부 국면에서 246까지).  
  - 레벨 8: 14~134 (초반 국면 14~44, 이후 82~134).
- **선택된 수의 visits**  
  - **레벨 30**: 대부분 **2**, 일부 4~133 → minVisits=2·폴백·짧은 예산 영향.  
  - **레벨 14**: 6~24 → 풀도 크고, 상대적으로 더 탐색된 수 선택.  
  - **레벨 8**: 2~6 → 풀은 6~8이지만, 시간 제한 안에서 2~6 방문 수도 선택됨.  
  - **레벨 28**(참고): 7~23.
